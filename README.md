## 🌟 Inspiration & Philosophy

Aura Reader is heavily inspired by Andrej Karpathy's [reader3](https://github.com/karpathy/reader3), reflecting a shared philosophy on how we should build tools to read alongside modern AI.

### The `reader3` Connection
Karpathy built `reader3` as a minimalist, self-hosted utility designed to break down long-form text chapter-by-chapter, optimizing the process of consuming books in tandem with LLMs. Rather than relying on rigid, bloated framework layers, it championed a fluid, lightweight approach where the code is simple, hackable, and completely self-contained.

Aura Reader adapts that exact thesis for deep, focused consumption:
* **The Death of Over-Engineering:** As Karpathy noted, *"Code is ephemeral now and libraries are over."* Instead of wrapping this reader in complex scaffolding, it is built entirely as a zero-dependency, single-file browser application.
* **LLM-First Co-Authoring:** If you want to change, expand, or adjust how your reader behaves, you don't need a heavy build environment. Because it is pure vanilla HTML, CSS, and JavaScript, you can simply feed this entire file back to an LLM, describe the feature you want, and watch it adapt instantly.
* **Maximizing the Input Loop:** Just as `reader3` optimized the human-AI loop for book reading, Aura Reader puts total control over your reading environment and context packaging directly back into your hands.

## 🧭 The Quick Start Map

Ready to escape the endless scroll and rebuild your reading muscle? Follow these steps to begin.

1. **Clone or Copy the Architecture:** (Under 30 seconds). Download or copy the raw `index.html` file into your local environment directory or visit the live deployment page.
2. **Launch the Desktop Environment:** (No server required). Double-click `index.html` directly from your file manager. It loads instantly right in the browser.
3. **Constructing Prompt Packages:** Highlight text anywhere in an active chapter block to call up the Context Bubble. Choose an operational modality to instantly construct structured system configurations (encapsulated within clean XML anchors) sent directly to your system clipboard.
4. **Configure Cognitive Shifting:** Toggle Quiet Eye Isolation or set your Optokinetic Tracking speed slider to hit your perfect reading flow state.
